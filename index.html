hdnom offers automatic parameter tuning and model selection functions for building 9 types of penalized Cox models with high-dimensional data.

Relevant functions:
hdcox.lasso, hdcox.alasso, hdcox.flasso, hdcox.enet, hdcox.aenet, hdcox.scad, hdcox.snet, hdcox.mcp, hdcox.mnet

Nomogram visualization is supported for all penalized Cox models built with the hdcox.*() functions.

Relevant functions:
hdnom.nomogram

Model validation could be done with bootstrap, k-fold cross-validation, and repeated k-fold cross-validation. Three types of time-dependent AUC measures are supported.

Relevant functions:
hdnom.validate, hdnom.external.validate

Model calibration by direct fitting, bootstrap, k-fold cross-validation, and repeated k-fold cross-validation, are supported in hdnom.

Relevant functions:
hdnom.calibrate, hdnom.external.calibrate

Methods provided

in hdnom

Print

Objects returned by hdnom.nomogram(), hdnom.validate(), and hdnom.calibrate() can be printed with their basic information.

Summary

Users could use summary() on objects returned by hdnom.validate() and hdnom.calibrate() to view the summary of the validation and calibration results.

Plot

Objects returned by hdnom.nomogram(), hdnom.validate(), and hdnom.calibrate() can be visualized with plot().

Model Building & Nomogram Plotting






Nomograms established an intuitive graphical representation for predictive models. It is a useful tool for evaluating risk and illustrating important variables for clinical prognoses.

In hdnom, the function hdnom.nomogram() helps us build nomograms for penalized Cox models built with hdcox.*() functions.

hdnom.nomogram()

model.type
Fitted model type. One of "lasso", "alasso", "flasso", "enet", "aenet", "mcp", "mnet", "scad", or "snet".
lambda
Value of the penalty parameter lambda. Selected variables are used at lambda to build the nomogram, and make predictions.
pred.at
Time point at which to plot nomogram prediction axis.

Model Validation





As a common practice, we usually employ resampling-based methods to validate the predictive performance of the regression model. Bootstrap, k-fold cross-validation, and repeated k-fold cross-validation are the most frequently utilized methods for such purpose.

The functions hdnom.validate() and hdnom.external.validate() allows users to assess model performance by time-dependent AUC (tAUC) with the three methods above.

R functions & key options

hdnom.validate()

model.type
Model type to validate. One of "lasso", "alasso", "flasso", "enet", "aenet", "mcp", "mnet", "scad", or "snet".
alpha
Value of the elastic-net mixing parameter. alpha=1: lasso (l1) penalty; alpha=0: ridge (l2) penalty. From the built Cox model.
lambda
Value of the penalty parameter lambda to use in the model fits on the resampled data. From the built Cox model.
pen.factor
Penalty factors to apply to each coefficient. From the built adaptive lasso or adaptive elastic-net model.
gamma
Value of the model parameter gamma for MCP/SCAD/Mnet/Snet models.
method
Validation method. Could be "bootstrap", "cv", or "repeated.cv".
boot.times
Number of repetitions for bootstrap.
nfolds
Number of folds for cross-validation and repeated cross-validation.
rep.times
Number of repeated times for repeated cross-validation.
tauc.type
Type of time-dependent AUC. Including "CD" proposed by Chambless and Diao (2006)., "SZ" proposed by Song and Zhou (2008)., "UNO" proposed by Uno et al. (2007).
tauc.time
Numeric vector. Time points at which to evaluate the time-dependent AUC.

Model Calibration





Measuring how far the model predictions are from true survival outcomes is known as model calibration. Calibration can be assessed by computing and plotting the predicted survival probabilities vs. actual observed survival probabilities.

Functions hdnom.calibrate() and hdnom.external.calibrate() offer 4 methods for model calibration, include direct fitting, bootstrap resampling, k-fold cross-validation and repeated cross-validation.

Model Comparison





Choosing an appropriate type of penalized Cox model is an essential step in survival modeling. To make things easier, we provided functions for comparing the emipirical predictive performance of different models.

Functions hdnom.compare.validate() and hdnom.compare.calibrate support model comparison by validation and calibration.

Kaplan-Meier Analysis of Risk Groups





Kaplan-Meier analysis is a necessary step for measuring the survival profiles for the risk groups identified by model calibration.

The function hdnom.kmplot() could help you conduct Kaplan-Meier analysis for the risk groups derived from the calibration results.

  • Example Kaplan-Meier Survival Curve

    Click the icon to zoom in

R functions & key options

hdnom.kmplot()

object
An object returned by hdnom.calibrate or hdnom.external.calibrate.
group.name
Risk group labels. Default is Group 1, Group 2, ..., Group k.
time.at
Time points to evaluate the number at risk.
col.pal
Color palette to use. Possible values are "JCO", "Lancet", "NPG", and "AAAS". Default is "JCO".
