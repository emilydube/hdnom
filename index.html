hdnom provides automatic parameter and model selection functions for building high-dimensional penalized Cox models.

Available functions:
hdcox.lasso, hdcox.alasso, hdcox.enet, hdcox.aenet

With hdnom, nomogram plotting is supported for all glmnet Cox model objects, from glmnet()` or hdcox.*() functions.

Available functions:
hdnom.nomogram

Time-dependent AUC based model validation is supported. Bootstrap, k-fold cross-validation, repeated k-fold cross-validation are all available.

Available functions:
hdnom.validate

Model calibration by direct fitting, bootstrap, k-fold cross-validation, repeated k-fold cross-validation, are supported in hdnom.

Available functions:
hdnom.calibrate

Methods provided

in hdnom

Print

Objects returned by hdnom.nomogram(), hdnom.validate(), and hdnom.calibrate() can be printed to view the basic modeling information of the object.

Summary

Users can use summary() on the objects returned by hdnom.validate() and hdnom.calibrate() to view the summary of the validation and calibration results.

Plot

To visualize the nomogram, validation and calibration results, objects returned by hdnom.nomogram(), hdnom.validate(), and hdnom.calibrate() can be plotted with plot().

Nomogram Plotting

Nomogram is a useful tool for evaluating risk and illustrating important variables for clinical prognoses. As an intuitive graphical representation for predictive models, nomograms reduce multi-variate regression models to a single numerical estimate of the probability of clinical events, such as overall survival.

The hdnom.nomogram() function helps us plot nomograms for penalized Cox models built with hdcox.*() functions and glmnet().

Fold numbers of cross-validation.

Model selection criterion, "lambda.min" or "lambda.1se".

hdcox.enet() / hdcox.aenet()

nfolds
Fold numbers of cross-validation.
alphas
Numeric vector. Alphas to tune, from 0 to 1.
rule
Model selection criterion, "lambda.min" or "lambda.1se".

hdnom.nomogram()

lambda
Value of the penalty parameter lambda in glmnet. We will use the selected variables at the provided lambda to build the nomogram, and make predictions.
pred.at
Time point at which to plot nomogram prediction axis.

Model Validation

As a common practice, we usually employ data splitting and resampling methods to validate the predictive performance of a regression model. Bootstrap, k-fold cross-validation, and repeated k-fold cross-validation are the most frequently utilized data splitting methods for such model performance validation purpose.

The hdnom.validate() function allows the user to assess the model performance by time-dependent AUC (tAUC) with the three listed data splitting and resampling methods.

R functions & key options

hdnom.validate()

alpha
Value of the elastic-net mixing parameter alpha in glmnet. alpha=1: lasso; alpha=0: ridge. From the Cox model you have built.
lambda
Value of the penalty parameter lambda to use in the glmnet fits on the resampled data. From the Cox model you have built.
pen.factor
Penalty factors to apply to each coefficient. From the adaptive penalized Cox model you have built.
method
Validation method. Could be "bootstrap", "cv", or "repeated.cv".
boot.times
Number of repetitions for bootstrap.
nfolds
Number of folds for cross-validation and repeated cross-validation.
rep.times
Number of repeated times for repeated cross-validation.
tauc.type
Type of time-dependent AUC. Including "CD" proposed by Chambless and Diao (2006)., "SZ" proposed by Song and Zhou (2008)., "UNO" proposed by Uno et al. (2007).
tauc.time
Numeric vector. Time points at which to evaluate the time-dependent AUC.

Model Calibration

Measuring how far the model predictions are from true survival outcomes is known as model calibration. Calibration can be assessed by computing and plotting the predicted probabilities from the model versus actual survival probabilities.

hdnom.calibrate() provides 4 available methods for model calibration, including direct fitting, bootstrap resampling, k-fold cross-validation and repeated cross-validation.

R functions & key options

hdnom.calibrate()

alpha
Value of the elastic-net mixing parameter alpha in glmnet. alpha=1: lasso; alpha=0: ridge. From the Cox model you have built.
lambda
Value of the penalty parameter lambda to use in the glmnet fits on the resampled data. From the Cox model you have built.
pen.factor
Penalty factors to apply to each coefficient. From the adaptive penalized Cox model you have built.
method
Calibration method. Options including "fitting", "bootstrap", "cv", and "repeated.cv".
boot.times
Number of repetitions for bootstrap.
nfolds
Number of folds for cross-validation and repeated cross-validation.
rep.times
Number of repeated times for repeated cross-validation.
pred.at
Time point at which calibration should take place.
ngroup
Number of groups to be formed for calibration.

about hdnom

Modeling high-dimensional data is not simple, but the modeling process should be easy and fun. To make the workflow of high-dimensional survival data analysis robust and reproducible enough, we built this package, hdnom.
